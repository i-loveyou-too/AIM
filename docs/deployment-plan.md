# Aim ON 배포 계획

최종 업데이트: 2026-03-31

## 현재 상황 요약
- 현재 상태는 **교사용 웹 목업 + DB 설계 완료** 단계다.
- 학생 몇 명 대상의 **소규모 테스트 배포**가 목표다.
- Next.js 기반 화면은 준비되어 있고, DB 스키마/시드/VIEW는 구성되어 있다.
- 아직 대규모 운영 전이므로 **복잡한 인프라보다 빠른 URL 확보**가 우선이다.

## 추천 배포 조합
- **프론트/웹앱: Vercel Hobby**
- **DB: Neon Free**

예시 배포 URL:
```txt
https://aim-on-xxxx.vercel.app
```

예시 DB 연결 문자열:
```txt
DATABASE_URL=postgresql://user:password@host/dbname?sslmode=require
```

## 왜 이 조합이 맞는지
- **무료로 바로 시작 가능**하고 초기 테스트에 충분하다.
- Vercel은 Next.js 배포가 빠르고 Git 연동/환경변수 관리가 쉽다.
- Neon Free는 카드 없이 시작 가능하고, 소규모 테스트에 필요한 성능/운영 기능(autoscaling, scale-to-zero, pooled connections)을 제공한다.
- 로컬 Postgres를 인터넷에 직접 노출하는 방식보다 **구성/보안/운영 난이도**가 낮다.

## 지금 이해해야 할 핵심
- DBeaver에서 실행한 SQL은 **로컬 Postgres**에 반영된 상태다.
- 학생이 실제 사용하려면 **배포된 웹 + 배포된 DB**가 필요하다.
- 즉, **로컬 DB만으로는 외부 학생 사용 불가**다.

## 배포 순서(step-by-step)
- [ ] 1단계: GitHub에 프로젝트 푸시
- [ ] 2단계: Neon 계정 생성 후 새 프로젝트 생성
- [ ] 3단계: Neon `DATABASE_URL` 복사
- [ ] 4단계: Neon DB에 SQL `01~07` 재실행
- [ ] 5단계: Vercel에 GitHub 저장소 연결
- [ ] 6단계: Vercel 환경변수에 `DATABASE_URL` 등록
- [ ] 7단계: 배포 후 URL 확인
- [ ] 8단계: 학생 테스트 대상에게 URL 공유

## 무료 플랜 가능 여부
- **지금 단계에서는 무료 플랜으로 충분**하다.
- Vercel Hobby: 무료 시작 가능
- Neon Free: 무료, 카드 불필요, 프로토타입/개발용 적합
- 대상이 소수 학생(파일럿)이라면 무료 범위 내 운영 가능성이 높다.

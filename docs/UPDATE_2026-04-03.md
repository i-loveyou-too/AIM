# Update 2026-04-03

## 목적
2026-04-03 기준으로 최근 반영된 로그인/프로필/설정/DB 연결 변경점을 한 번에 확인하기 위한 요약 문서다.

## 1. 로그인 후 프로필 반환 로직 수정
- 로그인 인증은 그대로 Django `auth_user`를 사용한다.
- 로그인 후 표시 정보는 `public.users`를 기준으로 조회한다.
- 보조 정보는 아래 순서로 연결한다.
  - `academy_members`
  - `teacher_profiles`
- 이름 반환 우선순위는 다음과 같다.
  1. `teacher_profiles.display_name`
  2. `users.name`
  3. `users.login_id`
  4. `users.email`

핵심 변경:
- `teacher_profiles` row가 없어도 `users.name`으로 이름이 반환되도록 수정
- 하드코딩 이름 매핑 제거
- `LEFT JOIN` 기반으로 보조 테이블 부재 시에도 `users` 정보가 살아 있도록 정리

## 2. 기존 계정 `lim` 연결 완료
기존 로그인 계정을 새 구조에 안전하게 연결했다.

현재 확인된 상태:
- `auth_user.username = lim`
- `users.login_id = lim`
- `users.name = test`
- `academy_members.academy_role = teacher`
- `teacher_profiles.display_name = test`
- `academies.name = Default Academy`

즉, `lim` 로그인 시 이름은 현재 `test`로 반환된다.

## 3. 교사 설정 화면 정리
- `src/app/dashboard/settings/page.tsx`는 API 응답 우선 렌더 구조를 유지한다.
- 아직 DB에 저장 구조가 없는 설정성 값은 화면에서 `"<db 데이터필요>"`로 표시하도록 수정했다.

현재 `"<db 데이터필요>"` 대상:
- 알림 설정
- 리포트 설정
- 수업 설정
- 과제 설정
- 반/시험 일정/커리큘럼 연결 중 아직 DB row가 없는 값

의도:
- 비어 있는 값을 의미 없는 기본값으로 숨기지 않기
- 실제 DB 입력 또는 테이블 연결이 필요한 지점을 명확히 드러내기
